IMS Research


Currently my github page is focused on software components created for my research into dynamic service orchestration in the IMS. All components bar the client are designed to work together to provide a system capable of making intelligent decisions about how an end user's services are triggered. Please note that all components should be treated as Alpha software, and will likely contain bugs. However, I believe it will prove a valuable starting point for further research in this area. All components were built according to my needs, but I am willing to improve them / make them more specification compliant should somebody find them useful.

SIP Library

To prototype the different components and client a custom SIP library was written. Various attempts were made at using pre-existing libraries, in C# and other languages, but limitations of these clients in a research environment (read only headers, automatically generated headers etc) lead me to create my own library.

Context Server (CS)

The CS was designed to be a central point for retrieving contextual information about the end user. The current implementation only supports presence at the moment but should be easy to extend to support additional context sources.

View details »

User Service Policy Server (USPS)

The USPS is how the system captures the end user's input. It allows them to view available services, and to create customised service flows. In other words, it allows them to explicitly detail under which conditions a particular service should be activated. This is done through an HTTP web interface.

View details »

Service Capability Interaction Manager (SCIM)

The Service Capability Interaction Manager acts as the intelligent routing agent in the proposed system, taking in information provided by the USPS and CS to route SIP messages to different destinations.

Service Repository Server (SRS)

The Service Repository Server (SRS) is used to hold information about the available services that are on offer in the network. With the proposed framework this information is trasmitted automatically to the SRS from each Application Server.
